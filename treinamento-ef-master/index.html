Treinamento - Eficiência Fiscal

Instruções

Nesta página você irá encontrar o treinamento básico para desenvolvedores da Eficiência Fiscal. Ele é composto de uma série de listas de exercícios que devem ser resolvidas em ordem, utilizando a linguagem de programação PHP em sua versão mais recente.

O uso da linguagem PHP é obrigatório e se justifica pelos seguintes pontos:

  1. É a nossa principal linguagem de desenvolvimento.
  2. É uma linguagem imperativa, bastante completa e de fácil aprendizado.
  3. É a linguagem da framework Laravel.

Apesar disso, o objetivo principal do treinamento não é ensinar PHP, de modo que deve ser seguido mesmo por quem já tenha conhecimento da linguagem.

Tente fazer os exercícios por conta própria o máximo que conseguir, mas não é proibido buscar ajuda dos colegas de trabalho, sobretudo se você ficar muito tempo travado em algum exercício em particular. Além disso, é recomendado mostrar o seu trabalho para os colegas, a fim de buscar opiniões e conselhos, e obrigatório submeter cada lista completa ao responsável pelo treinamento, para revisão.

Não comece uma nova lista antes de ter a revisão aprovada pelo responsável pelo treinamento -- tirando quando parecer razoável, por exemplo, caso tenha faltado apenas alguns ajustes pequenos. Definitivamente não antes do responsável ter sequer olhado a lista anterior. Se a revisão demorar, utilize o tempo para estudar.

Materiais complementares

Sinta-se livre para usar qualquer material que desejar para aprender PHP. O conteúdo recomendado para o treinamento pode ser encontrado na seção Referência da Linguagem do manual do PHP, em especial os seguintes itens:

Referência básica da linguagem

Introdução

O que segue é uma referência básica da linguagem, a título de material de apoio -- em especial no início do treinamento. A intenção não é ser a sua única referência de aprendizado.

Fique a vontade para apresentar sugestões de adições a esta seção.

Variáveis e execução

Todo arquivo PHP deve começar com <?php. Você declara variáveis utilizando o símbolo cifrão $, e as expressões terminam com ponto e vírgula ;: 

  <?php

  $numero = 1;
  $proximo = $numero + 1;

Para executar um programa, utilize o comando php:
php meuprograma.php

Condicionais

Um condicional utiliza as palavras if, else if e else: 

  <?php

  $numero = $10;

  if ($numero > 10) {
      echo "O número é maior do que 10";
  } else if ($numero < 10) {
      echo "O número é menor do que 10";
  } else {
      echo "O número é 10";
  }

O exemplo acima também mostra como imprimir um texto na tela através da palavra echo. A tabela a seguir resume os operadores condicionais:

Símbolo Exemplo Significado
> $a > $b $a é maior do que $b
< $a < $b $a é menor do que $b
>= $a >= $b $a é maior ou igual a $b
<= $a <= $b $a é menor ou igual a $b
== $a == $b $a é igual a $b
!= $a != $b $a é diferente de $b
! !$a $a é falso

A tabela a seguir resume algumas funções úteis e muito comuns em condicionais:

Função Exemplo Significado
empty empty($a) $a está vazio
is_null is_null($a) $a é null
instanceof $a instanceof Carro $a é um objeto da classe Carro

Arrays e iteração

Um array (lista) é declarado usando colchetes []: 

  <?php

  $a = [1, 2, 3];

  if (empty($a)) {
      echo "O array está vazio!";
  } else {
      echo "O primeiro item é " . $a[0];
  }

  $a[] = 4;
  $a[] = 5;

O exemplo anterior também mostra como adicionar uma variável a uma string (concatenação) através do operador ponto, e como adicionar valores ao fim do array. Você pode iterar através dos itens de uma lista de diversas maneiras: 

  <?php

  $a = [1, 2, 3];

  foreach ($a as $numero) {
      echo "$numero";
  }

  for ($i = 0; $i < count($a); $a++) {
      echo($a[$i]);
  }

  $i = 0;
  while ($i < count($a)) {
      echo($a[$i]);
      $i++;
  }

Controle de iteração

Você pode sair de um loop utilizando a palavra break, e pular para a próxima iteração utilizando a palavra continue.

Arrays associativos

Um array pode ser associativo, isto é, indexado por palavras, e não somente números: 

  <?php

  $a = [
      'nome' => 'João',
      'idade' => 15
  ];

  echo "O seu nome é : " . $a['nome'];

  $a['naturalidade'] = 'Brasileiro';

Leitura de input do usuário

No PHP, você pode ler /input/ do usuário através da função read_line: 

  <?php

  $numero = readline("Digite um número: ");
  echo "Você escolheu: " . $numero;

Conversão entre tipos

Você pode converter uma string em um inteiro através da sintaxe (int): 

  <?php

  $numero = (int) readline("Digite um número: ");

  if ($numero > 10) {
      echo "Você escolheu um número grande!";
  }

Números aleatórios

Você pode gerar um número aleatório através da função random_int: 

  <?php

  echo(random_int(0, 10));

Funções

Você declara funções da seguinte maneira:
function <nome>(<parametro1>, <parametro2>...) { corpo } 

  function adicionarAspas($str, $simples=false) {
      if ($simples) {
	  return "'{$str}'";
      } else {
	  return "\"{$str}\"";
      }
  }

O exemplo acima também mostra como adicionar um valor padrão para um parâmetro.

Interação com o sistema operacional

Você pode interagir com o sistema operacional através da função system. É muito útil, por exemplo, utilizar system('clear') para limpar o texto do terminal no Linux.

Lista 1: Jogo da adivinhação

A recomendação é fazer os exercícios desta lista na sequência, mesmo que pareça mais fácil fazer em ordem diversa.

  1. Escreva um jogo de adivinhação: o usuário precisa adivinhar um número entre 0 e 100, e o programa deve dizer, a cada tentativa, se o usuário chutou muito alto, muito baixo, ou se acertou.
  2. Incremente o programa anterior para exibir uma pontuação ao final da partida. Quanto menos chutes o usuário precisar para acertar, maior deve ser a sua pontuação.
  3. Ofereça, ao fim do jogo, a possibilidade do usuário jogar novamente.
  4. Exiba uma mensagem de congratulação quando o usuário atingir a pontuação mais alta até então.
  5. Quando o usuário não atingir a pontuação mais alta, exiba uma mensagem de "Game Over" com a pontuação dele e a pontuação mais alta.
  6. Ofereça duas opções ao usuário: iniciar o jogo ou visualizar o ranking de pontuação.
  7. Pergunte o nome do jogador no início da partida e associe o nome dele à pontuação (no ranking, por exemplo).
  8. Desafio: salve o ranking entre as partidas, mesmo que o programa seja fechado e aberto de novo.
  9. Desafio: faça o tempo ser um dos fatores da pontuação -- quanto mais rápido o usuário acertar, maior será a sua pontuação.
  10. Considere a pontuação por jogador, exibindo mensagens como "você atingiu a sua maior pontuação" e "você atingiu a maior pontuação da máquina".

Lista 2: Jogo da forca

  1. Escreva um jogo da forca: o sistema escolhe uma palavra aleatória e exibe a quantidade de letras dela. O usuário tem uma quantidade limitada de vidas (tentativas erradas) para adivinhar a palavra inteira.
  2. Faça uma representação visual da situação da palavra. Por exemplo: tracinhos para representar as letras ainda não adivinhadas, e as letras em si nos lugares corretos.
  3. Exiba uma lista das letras que o usuário já utilizou, assim como a quantidade de vidas restantes.
  4. Separe as palavras possíveis em grupos, e exiba uma dica correspondente para o jogador. Por exemplo: animal, cidade, fruta.
  5. Permita que palavras novas sejam cadastradas pelo jogador antes de iniciar uma partida.
  6. Permita que grupos novos sejam cadastrados pelo jogador antes de iniciar a partida.
  7. Utilize palavras compostas com espaço na sua lista de palavras, por exemplo: São Paulo, Arco e flecha, Futebol de botão. Garanta que elas sejam exibidas adequadamente, isto é, não deve existir um tracinho no lugar dos espaços. Para São Paulo, por exemplo: 
     _ _ _  _ _ _ _ _
  8. Garanta que, se o usuário digitar "a" para a palavra São Paulo, as duas vogais sejam marcadas: 
    _ ã _  _ a _ _ _
  9. Desafio: exiba um desenho da forca (pode ser um desenho feito com caracteres) de acordo com a quantidade de vidas restantes
  10. Desafio: permita que dois jogadores joguem ao mesmo tempo, um contra o outro -- vence quem adivinhar a palavra.

Lista 3: Trabalhando com coleções

Para os exercícios desta lista, evite utilizar funções que já existem na biblioteca padrão do PHP. Por exemplo, não utilize a função max para resolver o primeiro exercício. Em vez disso, crie a sua própria função utilizando os elementos básicos da linguagem PHP.

A sequência dos exercícios desta lista é apenas uma sugestão, do mais fácil ao mais difícil, mas pode ser feita em qualquer ordem desejada.

O formato de entrega deve ser um arquivo com as definições das funções dos exercícios, pelo menos uma para cada exercício. A função obrigatoriamente recebe como parâmetro um array e retorna (return ...;) o que estiver descrito em "saída". A função deve funcionar com qualquer array de input razoável, sendo que a "entrada" e a "saída" são apenas exemplos.

  1. Escreva uma função que receba um array de valores numéricos e retorne o valor mais alto.

    entrada: [1, 5, 7, 8, 1, 20]
    saída: 20
  2. Escreva uma função que receba um array de valores numéricos e retorne a soma dos valores.

    entrada: [1, 5, 7, 1]
    saída: 14
  3. Escreva uma função que receba dois arrays e retorne um array de valores intercalados.

    entrada: [1, 5, 7, 1]
    saída: 14
  4. Escreva uma função como a anterior, mas que retorne um array de pares.

    entrada: [1, 2, 3, 4], ['a', 'b', 'c', 'd']
    saída: [[1, 'a'], [2, 'b'], [3, 'c'], [4, 'd']]
  5. Escreva uma função que embaralhe um array.

    entrada: [1, 2, 3, 4, 5]
    saída: [5, 2, 4, 1, 3]
  6. Escreva uma função que receba um array associativo e um array de strings, e retorne uma versão do primeiro array somente com as chaves do segundo.

    entrada: ['nome' => 'Jacó', 'idade' => 74, 'profissão' => 'ancião'], ['nome', 'profissão']
    saída: ['nome' => 'Jacó', 'profissão' => 'ancião']
  7. Escreva uma função que ordene um array.

    entrada: [1, 5, 2, 4, 3]
    saída: [1, 2, 3, 4, 5]
  8. Escreva uma função que remova valores duplicados de um array.

    entrada: [1, 2, 3, 3, 4, 5, 4]
    saida: [1, 2, 3, 4, 5]
  9. Escreva uma função que reverta um array.

    entrada: [1, 2, 3, 4, 5]
    saída: [5, 4, 3, 2, 1]
  10. Desafio: escreva uma função que achate um array multidimensional.

    entrada: [1, [1, 2], [1, [2, 3], 4]]
    saída: [1, 1, 2, 1, 2, 3, 4]

Lista 4: trabalhando com o banco de dados

Para essa lista de exercícios, vamos precisar de um banco de dados instalado na máquina. Eu recomendo o SQLite3 no Linux, e o MySQL no Windows, por questão de facilidade de instalação.

  1. Instale o banco de dados na sua máquina e configure a conexão a partir do PHP.
  2. Crie uma aplicação para controlar o estoque de uma papelaria. O usuário precisa conseguir cadastrar os itens do estoque junto da quantidade inicial de cada item, e depois registrar as entradas e saídas de cada item. A qualquer momento, ele precisa conseguir gerar um relatório da situação atual do estoque, além de consultar a situação de algum item específico. Esse relatório precisa ser salvo num arquivo ou impresso em tela, a escolha do usuário.
  3. Guarde um registro de cada movimentação do estoque. Permita que o usuário gere um relatório da movimentação do estoque de determinado item.
  4. Permita que o usuário gere um relatório da movimentação do estoque por período.

Projeto final

Para a sua graduação, você terá que desenvolver um sistema WEB de sua escolha, com os seguintes requisitos:

  • CRUD de pelo menos um recurso
  • Impressão de pelo menos um relatório
  • Não pode ter autenticação, isto é, não pode ter login e nem permissões
  • Precisa resolver algum problema real, mesmo que em escala reduzida
Deve ser obrigatoriamente desenvolvido com as seguintes tecnologias:
  • Laravel na versão mais recente
  • Bootstrap 4

Você não pode criar uma Single Page Application, isto é, nada de utilizar Vue, React e companhia. As páginas devem ser renderizadas no back-end, utilizando a linguagem de layout Blade, do próprio Laravel. Javascript só pode ser utilizado para melhoria progressiva (progressive enhancement) da página: quer dizer que ela precisa funcionar (pelo menos num nível básico) mesmo sem JavaScript, mas pode ter funcionalidades que melhoram a experiência do usuário quando o JavaScript estiver ativado, como busca em tempo real, efeitos visuais etc..

É recomendado o uso da biblioteca JQuery para a manipulaçào da página com JavaScript.

Antes de começar, valide o seu projeto com o responsável pelo treinamento. Uma vez que a aplicação inicial do Laravel esteja configurada e conectada com o banco de dados, aguarde uma aula introdutória de desenvolvimento WEB com o responsável pelo treinamento antes de iniciar o desenvolvimento do projeto. Enquanto isso, vá lendo a documentação do Laravel, que é excelente, ou reforçando os conhecimentos básicos de HTML e CSS.

Orientação importante: ao contrário das listas de exercicios, é recomendado que sejam feitas orientações regulares no decorrer do desenvolvimento do projeto final, e não apenas na revisão final. Possivelmente uma vez ao dia, ou uma vez a cada dois dias. Então faça o melhor que puder por conta própria, mas organize suas dúvidas e prepare-se para apresentar o andamento do projeto regularmente.

O prazo esperado para conclusão desta etapa final do treinamento é entre uma e duas semanas, mas desvios podem ocorrer. Em caso de dúvida, converse com o responsável pelo treinamento.
